使用 Nocalhost 开发 APISIX Ingress Controller

1. 准备工作#

APISIX Ingress Controller 可以部署在单节点的或托管的 Kubernetes 集群上。 您可以在 Docker Desktop、Minikube 或公有云平台（如 GKE、AWS EKS、AKS 等）上部署它。

本文将在 Docker Desktop 上部署 APISIX Ingress Controller，并使用 Nocalhost 插件在 GoLand 中演示如何开发 APISIX Ingress Controller。

2. 安装 APISIX Ingress Controller#

在 Goland 上安装了 Nocalhost 插件后，进入一个集群，选择一个准备安装 APISIX Ingress Contoller 的命名空间；右键点击命名空间，然后在弹出菜单上点击 Install Application。 在出现的对话框中，Name 文本框中输入 apisix-ingress-controller，然后在 Chart URL 中输入 https://charts.apiseven.com。 其他选项可以使用默认设置。

3. 开发 APISIX Ingress Controller#

右键点击 apisix-ingress-controller，并在菜单上点击 Dev Config。 复制并粘贴下面的 Nocalhost 配置内容到刚刚打开的文件，然后保存文件。

name: apisix-ingress-controllerserviceType: deploymentcontainers:  - name: apisix-ingress-controller    dev:        gitUrl: https://github.com/apache/apisix-ingress-controller.git        image: codingcorp-docker.pkg.coding.net/nocalhost/dev-images/golang:1.16        shell: ""        workDir: /home/nocalhost-dev        storageClass: ""        resources: null        persistentVolumeDirs: []        command:            build: []            run:              - go              - run              - main.go              - ingress              - --config-path              - conf/config-default.yaml            debug:              - dlv              - --headless              - --log              - --listen              - :2345              - --api-version              - "2"              - --accept-multiclient              - debug              - main.go              - ingress              - --config-path              - conf/config-default.yaml            hotReloadRun: []            hotReloadDebug: []        debug: null        useDevContainer: false        sync: null        env: []        envFrom: null        portForward:          - 8080:8080

右键点击 apisix-ingress-controller 部署项，然后在菜单上点击 Start DevMode。 DevMode 启动后，点击 GoLand 上的 Run/Debug 按钮，然后您可以通过访问 http://127.0.1:8080 在浏览器上查看 APISIX 仪表盘界面。

接下来，您可以继续享受 APISIX Ingress Controller 的开发及调试过程。
